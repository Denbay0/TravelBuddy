# TravelBuddy

**TravelBuddy** — full-stack веб-приложение для путешественников: здесь можно **создавать маршруты**, **делиться впечатлениями постами**, **сохранять понравившееся**, **искать контент по всему приложению** и **генерировать PDF-отчёты по маршрутам**.  
Проект сделан как компактная соц-платформа вокруг путешествий с акцентом на удобный UI и безопасную cookie-авторизацию.

---

## Зачем проект

TravelBuddy закрывает типичный сценарий путешественника:

- собрать/описать маршрут (города, длительность, транспорт);
- сохранять идеи и «планы на потом»;
- делиться опытом в формате постов по городам;
- быстро находить людей/маршруты/посты через глобальный поиск;
- выгружать маршрут в **PDF** (например, чтобы отправить друзьям или сохранить офлайн).

---

## Ключевые возможности

### Маршруты
- Создание и редактирование маршрутов (название, описание, список городов, длительность, транспорт).
- Просмотр отдельного маршрута и подборка **trending** (популярные).
- **Сохранение маршрутов** в избранное.

### Сообщество (посты)
- Публикации о поездках: заголовок, текст, город, транспорт.
- **Лайки** и **сохранения** постов.
- **Комментарии** к постам.

### Профиль
- Редактирование профиля (описание, домашний город, предпочтительный транспорт и т.д.).
- Загрузка/удаление **аватара**.
- Быстрый доступ к своим постам и избранным маршрутам.

### Поиск
- Единый поиск по **маршрутам, постам и пользователям**.

### Отчёты
- Генерация **PDF-отчёта по маршруту** (удобно для шаринга/печати).

## Технологии

### Backend
- **FastAPI**
- **SQLAlchemy**
- **PostgreSQL**
- (опционально) **Redis**
- PDF: **ReportLab**
- Auth: **JWT + CSRF**, cookie-based сессия

### Frontend
- **React + TypeScript**
- **Vite**
- **Tailwind CSS**
- UI/UX: **framer-motion**
- Карта: **Leaflet + react-leaflet**
- Иконки: **lucide-react**
- Маршрутизация: **react-router-dom**

---

## Архитектура (кратко)

- `app/` — backend API (роуты, модели, схемы, безопасность).
- `web/` — frontend (страницы, компоненты, контекст авторизации, карта).
- Docker / compose-конфиги — для удобного окружения разработки/деплоя (не обязательно для понимания проекта).

---

## Основные сущности домена

- **User** — профиль путешественника (username/email/handle, био, город, теги, аватар).
- **Route** — маршрут (title/description/cities/duration/transport).
- **Post** — публикация (title/content/city/transport).
- **RouteSave / PostSave** — сохранённые маршруты/посты.
- **PostLike** — лайки.
- **PostComment** — комментарии.

---

## Безопасность

- Cookie-авторизация на основе **JWT**.
- Отдельная защита мутаций через **CSRF-токен**.
- Приватные действия (редактирование, лайк/сейв, профиль) доступны только авторизованным.

---

## API (что есть на бэке)

- `auth`: регистрация/логин/логаут, `me`, CSRF.
- `profile`: профиль “me”, аватар, мои посты, избранные маршруты.
- `routes`: CRUD маршрутов, trending, сохранить/убрать из избранного, preview.
- `posts`: CRUD постов, лайки/сейвы, комментарии.
- `search`: глобальный поиск.
- `reports`: генерация PDF по маршруту.
